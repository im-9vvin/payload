## 일반적인 문제

### 로그인을 시도할 때 "Unauthorized, you must be logged in to make this request" 오류

이는 로그인 시 인증 쿠키가 올바르게 설정되지 않거나 승인되지 않는다는 의미입니다. 해결하려면 Payload 구성에서 다음 설정을 확인하세요:

- CORS - '\*'를 사용하고 있다면, 지정한 도메인을 포함하여 특정 도메인만 명시적으로 허용해 보세요.
- CSRF - 이것이 설정되어 있나요? 그렇다면 도메인이 csrf 도메인 내에서 화이트리스트에 있는지 확인하세요. 그렇지 않다면 아마도 문제가 아니겠지만, 어쨌든 화이트리스트에 추가해도 해가 되지 않을 것입니다.
- 쿠키 설정. 이것들이 완전히 정의되지 않았다면 괜찮습니다. 하지만 쿠키 도메인이 설정되어 있거나 비슷한 것이 있다면, 도메인이 잘못 구성되지 않았는지 확인하세요.

이 오류는 로그인에 성공한 후 Payload가 설정하는 인증 쿠키가 구성 오류로 인해 거부되고 있다는 의미일 가능성이 높습니다.

문제를 추가로 조사하려면:

- 로그인 화면으로 이동하세요. 검사기를 열고 네트워크 탭으로 이동하세요.
- 로그인한 다음 네트워크 패널에 나타나야 하는 로그인 요청을 찾으세요. 로그인 요청을 클릭하세요.
- 로그인 요청은 응답에 Set-Cookie 헤더를 가져야 하고, 쿠키가 성공적으로 설정되어야 합니다. 그렇지 않다면 대부분의 브라우저에서 일반적으로 작은 노란색 ⚠️ 기호가 있어 마우스를 올리면 쿠키가 거부된 이유를 볼 수 있습니다.
