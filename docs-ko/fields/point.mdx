포인트 필드는 데이터베이스에 좌표 쌍을 저장하고 위치 관련 쿼리를 위한 인덱스를 할당합니다. 데이터베이스의 데이터 구조는 포인트를 나타내는 GeoJSON 구조와 일치합니다. Payload API는 객체 데이터를 [경도, 위도] 위치로만 단순화합니다.

<LightDarkImage
  srcLight="https://payloadcms.com/images/docs/fields/point.png"
  srcDark="https://payloadcms.com/images/docs/fields/point-dark.png"

  caption="포인트 필드의 관리자 패널 스크린샷"
/>

포인트 필드를 추가하려면 [필드 Config](./overview)에서 `type`을 `point`로 설정하세요:

```ts
import type { Field } from 'payload'

export const MyPointField: Field = {
  // ...
  type: 'point', // highlight-line
}
```

<Banner type="warning">
  **중요:** 포인트 필드는 현재 SQLite에서 지원되지 않습니다.
</Banner>

## 구성

| 옵션                   | 설명                                                                                                                                                                      |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`name`** \*          | 데이터베이스에 저장되고 검색될 때 속성 이름으로 사용됩니다. [자세한 내용](/docs/fields/overview#field-names).                                                            |
| **`label`**            | 관리자 패널에서 필드 레이블로 사용되고 생성된 GraphQL 타입의 이름을 지정합니다.                                                                                           |
| **`unique`**           | 컬렉션의 각 항목이 이 필드에 대해 고유한 값을 갖도록 강제합니다.                                                                                                          |
| **`index`**            | 더 빠른 쿼리를 생성하기 위해 이 필드에 대한 [인덱스](../database/indexes)를 구축합니다. 위치 쿼리를 지원하기 위해 포인트 인덱스는 기본적으로 `2dsphere`이며, 인덱스를 비활성화하려면 `false`로 설정하세요. |
| **`validate`**         | 관리자 패널과 백엔드 모두에서 실행될 커스텀 유효성 검사 함수를 제공합니다. [자세한 내용](/docs/fields/overview#validation).                                              |
| **`saveToJWT`**        | 이 필드가 최상위 수준이고 [인증](/docs/authentication/overview)을 지원하는 구성에 중첩된 경우 해당 데이터를 사용자 JWT에 포함합니다.                                    |
| **`hooks`**            | 이 필드에 대한 로직을 제어하기 위해 필드 훅을 제공합니다. [자세한 내용](../hooks/fields).                                                                                  |
| **`access`**           | 사용자가 이 필드의 데이터로 무엇을 보고 할 수 있는지 나타내는 필드 액세스 제어를 제공합니다. [자세한 내용](../access-control/fields).                                    |
| **`hidden`**           | 모든 API에서 이 필드의 가시성을 완전히 제한합니다. 데이터베이스에는 여전히 저장되지만 API나 관리자 패널에는 표시되지 않습니다.                                           |
| **`defaultValue`**     | 이 필드의 기본값으로 사용할 데이터를 제공합니다. [자세한 내용](/docs/fields/overview#default-values).                                                                    |
| **`localized`**        | 이 필드에 대해 현지화를 활성화합니다. 기본 구성에서 [현지화가 활성화](/docs/configuration/localization)되어야 합니다.                                                    |
| **`required`**         | 이 필드에 값이 있어야 합니다.                                                                                                                                             |
| **`admin`**            | 관리자 특정 구성. [자세한 내용](./overview#admin-options).                                                                                                                |
| **`custom`**           | 커스텀 데이터 추가를 위한 확장 지점 (예: 플러그인용)                                                                                                                      |
| **`typescriptSchema`** | JSON 스키마 제공으로 필드 유형 생성을 재정의                                                                                                                              |
| **`virtual`**          | 데이터베이스에서 필드를 비활성화하려면 `true`를 제공하거나 [관계와 필드를 연결](/docs/fields/relationship#linking-virtual-fields-with-relationships)하기 위해 문자열 경로를 제공합니다. [가상 필드](https://payloadcms.com/blog/learn-how-virtual-fields-can-help-solve-common-cms-challenges) 참조 |

_\* 별표는 필수 속성을 나타냅니다._

## 예제

`collections/ExampleCollection.ts`

```ts
import type { CollectionConfig } from 'payload'

export const ExampleCollection: CollectionConfig = {
  slug: 'example-collection',
  fields: [
    {
      name: 'location',
      type: 'point',
      label: '위치',
    },
  ],
}
```

## 쿼리 - near

다른 포인트까지의 거리를 기반으로 쿼리를 수행하려면 `near` 연산자를 사용할 수 있습니다. near 연산자를 사용하여 쿼리할 때 반환된 문서는 가장 가까운 것부터 정렬됩니다.

## 쿼리 - within

GeoJSON에서 정의된 특정 영역 내에 포인트가 있는지 여부를 기반으로 쿼리를 수행하려면 `within` 연산자를 사용할 수 있습니다.
예제:

```ts
const polygon: Point[] = [
  [9.0, 19.0], // 왼쪽 하단
  [9.0, 21.0], // 왼쪽 상단
  [11.0, 21.0], // 오른쪽 상단
  [11.0, 19.0], // 오른쪽 하단
  [9.0, 19.0], // 다각형을 닫기 위해 시작점으로 돌아가기
]

payload.find({
  collection: 'points',
  where: {
    point: {
      within: {
        type: 'Polygon',
        coordinates: [polygon],
      },
    },
  },
})
```

## 쿼리 - intersects

GeoJSON에서 정의된 특정 영역과 포인트가 교차하는지 여부를 기반으로 쿼리를 수행하려면 `intersects` 연산자를 사용할 수 있습니다.
예제:

```ts
const polygon: Point[] = [
  [9.0, 19.0], // 왼쪽 하단
  [9.0, 21.0], // 왼쪽 상단
  [11.0, 21.0], // 오른쪽 상단
  [11.0, 19.0], // 오른쪽 하단
  [9.0, 19.0], // 다각형을 닫기 위해 시작점으로 돌아가기
]

payload.find({
  collection: 'points',
  where: {
    point: {
      intersects: {
        type: 'Polygon',
        coordinates: [polygon],
      },
    },
  },
})
```

## 커스텀 컴포넌트

### 필드

#### 서버 컴포넌트

```tsx
import type React from 'react'
import { PointField } from '@payloadcms/ui'
import type { PointFieldServerComponent } from 'payload'

export const CustomPointFieldServer: PointFieldServerComponent = ({
  clientField,
  path,
  schemaPath,
  permissions,
}) => {
  return (
    <PointField
      field={clientField}
      path={path}
      schemaPath={schemaPath}
      permissions={permissions}
    />
  )
}
```

#### 클라이언트 컴포넌트

```tsx
'use client'
import React from 'react'
import { PointField } from '@payloadcms/ui'
import type { PointFieldClientComponent } from 'payload'

export const CustomPointFieldClient: PointFieldClientComponent = (props) => {
  return <PointField {...props} />
}
```

### 레이블

#### 서버 컴포넌트

```tsx
import React from 'react'
import { FieldLabel } from '@payloadcms/ui'
import type { PointFieldLabelServerComponent } from 'payload'

export const CustomPointFieldLabelServer: PointFieldLabelServerComponent = ({
  clientField,
  path,
}) => {
  return (
    <FieldLabel
      label={clientField?.label || clientField?.name}
      path={path}
      required={clientField?.required}
    />
  )
}
```
