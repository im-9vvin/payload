Collection Hooks는 특정 [Collection](../configuration/collections) 내의 Documents에서 실행되는 [Hooks](./overview)입니다. Document 라이프사이클의 특정 이벤트 중에 자신만의 로직을 실행할 수 있게 해줍니다.

Collection에 Hooks을 추가하려면 [Collection Config](../configuration/collections)에서 `hooks` 속성을 사용하세요:

```ts
import type { CollectionConfig } from 'payload'

export const CollectionWithHooks: CollectionConfig = {
  // ...
  hooks: {
    // highlight-line
    // ...
  },
}
```

<Banner type="info">
  **팁:** 특정 필드에 hook 로직을 격리시키기 위해 필드 수준에서도 hooks을 설정할 수 있습니다.
  [자세한 내용](./fields).
</Banner>

## Config Options

모든 Collection Hooks는 [동기 또는 비동기 함수](./overview#async-vs-synchronous) 배열을 받습니다. 각 Collection Hook은 자신만의 타입에 기반한 특정 인수를 받으며, 특정 출력을 수정할 수 있는 능력을 가집니다.```ts
import type { CollectionConfig } from 'payload';

export const CollectionWithHooks: CollectionConfig = {
  // ...
  // highlight-start
  hooks: {
    beforeOperation: [(args) => {...}],
    beforeValidate: [(args) => {...}],
    beforeDelete: [(args) => {...}],
    beforeChange: [(args) => {...}],
    beforeRead: [(args) => {...}],
    afterChange: [(args) => {...}],
    afterRead: [(args) => {...}],
    afterDelete: [(args) => {...}],
    afterOperation: [(args) => {...}],
    afterError: [(args) => {....}],

    // Auth-enabled Hooks
    beforeLogin: [(args) => {...}],
    afterLogin: [(args) => {...}],
    afterLogout: [(args) => {...}],
    afterRefresh: [(args) => {...}],
    afterMe: [(args) => {...}],
    afterForgotPassword: [(args) => {...}],
    refresh: [(args) => {...}],
    me: [(args) => {...}],
  },
  // highlight-end
}
```

### beforeOperation

`beforeOperation` hook은 작업이 받는 인수를 수정하거나 작업이 시작되기 전에 실행되는 부수 효과를 실행하는 데 사용할 수 있습니다.

사용 가능한 Collection 작업에는 `create`, `read`, `update`, `delete`, `login`, `refresh` 및 `forgotPassword`가 포함됩니다.```ts
import type { CollectionBeforeOperationHook } from 'payload'

const beforeOperationHook: CollectionBeforeOperationHook = async ({
  args,
  operation,
  req,
}) => {
  return args // 필요에 따라 수정된 작업 인수 반환
}
```

다음 인수들이 `beforeOperation` hook에 제공됩니다:

| Option           | Description                                                                                                                                           |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`collection`** | 이 Hook이 실행되고 있는 [Collection](../configuration/collections)입니다.                                                                 |
| **`context`**    | Hooks 간에 전달되는 사용자 정의 컨텍스트입니다. [자세한 내용](./context).                                                                                       |
| **`operation`**  | 이 hook이 실행되고 있는 작업의 이름입니다.                                                                                           |
| **`req`**        | [Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request) 객체입니다. [Local API](../local-api/overview) 작업의 경우 모의로 처리됩니다. |

### beforeValidate

`create` 및 `update` 작업 중에 실행됩니다. 이 hook을 사용하면 들어오는 데이터가 서버 측에서 유효성 검사되기 전에 데이터를 추가하거나 형식을 지정할 수 있습니다.

이것은 클라이언트 측 유효성 검사 전에는 실행되지 않는다는 점에 유의하세요. 프론트엔드에서 사용자 정의 필드 구성 요소를 렌더링하고 `validate` 함수를 제공하는 경우, 유효성 검사가 실행되는 순서는 다음과 같습니다:

1. `validate`가 클라이언트에서 실행됩니다
2. 성공하면 `beforeValidate`가 서버에서 실행됩니다
3. `validate`가 서버에서 실행됩니다

```ts
import type { CollectionBeforeValidateHook } from 'payload'

const beforeValidateHook: CollectionBeforeValidateHook = async ({ data }) => {
  return data
}
```다음 인수들이 `beforeValidate` hook에 제공됩니다:

| Option            | Description                                                                                                                                           |
| ----------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`collection`**  | 이 Hook이 실행되고 있는 [Collection](../configuration/collections)입니다.                                                                 |
| **`context`**     | Hooks 간에 전달되는 사용자 정의 컨텍스트입니다. [자세한 내용](./context).                                                                                       |
| **`data`**        | 작업을 통해 전달되는 들어오는 데이터입니다.                                                                                                       |
| **`operation`**   | 이 hook이 실행되고 있는 작업의 이름입니다.                                                                                           |
| **`originalDoc`** | 변경이 적용되기 전의 Document입니다.                                                                                                              |
| **`req`**         | [Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request) 객체입니다. [Local API](../local-api/overview) 작업의 경우 모의로 처리됩니다. |

### beforeChange

유효성 검사 직전에 beforeChange hooks는 create 및 update 작업 중에 실행됩니다. 이 단계에서 데이터는 유효성이 검증되지 않은 사용자 입력으로 처리되어야 합니다. 필수 필드가 존재하거나 필드가 올바른 형식인지에 대한 보장은 없습니다. 따라서 이 데이터를 부수 효과에 사용하려면 수동 유효성 검사가 필요합니다. 선택적으로 저장될 데이터의 모양을 수정할 수 있습니다.

```ts
import type { CollectionBeforeChangeHook } from 'payload'

const beforeChangeHook: CollectionBeforeChangeHook = async ({ data }) => {
  return data
}
```

| **`data`**        | 작업을 통해 전달되는 들어오는 데이터입니다.                                                                                                       |
| **`operation`**   | 이 hook이 실행되고 있는 작업의 이름입니다.                                                                                           |
| **`originalDoc`** | 변경이 적용되기 전의 Document입니다.                                                                                                              |
| **`req`**         | [Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request) 객체입니다. [Local API](../local-api/overview) 작업의 경우 모의로 처리됩니다. |

### afterError

`afterError` Hook은 Payload 애플리케이션에서 오류가 발생할 때 트리거됩니다. 이는 타사 서비스에 오류를 로깅하거나, 개발팀에 이메일을 보내거나, Sentry나 DataDog에 오류를 로깅하는 등에 유용할 수 있습니다. 출력은 결과 객체/상태 코드를 변환하는 데 사용할 수 있습니다.

```ts
import type { CollectionAfterErrorHook } from 'payload';

const afterErrorHook: CollectionAfterErrorHook = async ({
  req,
  id,
  doc,
}) => {...}
```다음 인수들이 `afterError` Hook에 제공됩니다:

| Argument            | Description                                                                                                                                                                                     |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`error`**         | 발생한 오류입니다.                                                                                                                                                                        |
| **`context`**       | Hooks 간에 전달되는 사용자 정의 컨텍스트입니다. [자세한 내용](./context).                                                                                                                                 |
| **`graphqlResult`** | GraphQL 결과 객체로, hook이 GraphQL 컨텍스트 내에서 실행되는 경우 사용 가능합니다.                                                                                                          |
| **`req`**           | [Web Request](https://developer.mozilla.org/en-US/docs/Web/API/Request)를 확장한 `PayloadRequest` 객체입니다. 현재 인증된 `user`와 Local API 인스턴스 `payload`를 포함합니다. |
| **`collection`**    | 이 Hook이 실행되고 있는 [Collection](../configuration/collections)입니다.                                                                                                           |
| **`result`**        | 포맷된 오류 결과 객체로, hook이 REST 컨텍스트에서 실행되는 경우 사용 가능합니다.                                                                                                       |
