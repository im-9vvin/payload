![https://www.npmjs.com/package/@payloadcms/plugin-seo](https://img.shields.io/npm/v/@payloadcms/plugin-seo)

이 플러그인을 사용하면 [관리자 패널](../admin/overview) 내에서 애플리케이션의 SEO 메타데이터를 쉽게 관리할 수 있습니다. [컬렉션](../configuration/collections) 및 [글로벌](../configuration/globals)에서 활성화되면 기본적으로 `title`, `description`, `image`가 포함된 새로운 `meta` 필드 그룹을 추가합니다. 프론트엔드 애플리케이션은 이 데이터를 사용하여 애플리케이션이 요구하는 방식으로 메타 태그를 렌더링할 수 있습니다. 예를 들어, `meta.title`을 콘텐츠로 사용하여 페이지의 `<head>`에 `title` 태그를 주입할 수 있습니다.

사용자가 관리자 패널 내에서 문서를 편집할 때 이러한 필드를 "자동 생성"하는 옵션이 있습니다. 클릭하면 이 플러그인은 제목, 설명 및 이미지를 재생성하는 사용자 정의 함수를 실행합니다. 이렇게 하면 애플리케이션에 자체 SEO 작성 지원을 직접 구축할 수 있습니다. 예를 들어, 페이지 제목에 사이트 이름을 추가하거나 문서의 발췌 필드를 설명으로 사용하거나 AI를 사용하여 이미지를 생성하는 타사 API와 통합할 수도 있습니다.

페이지가 검색 엔진에서 어떻게 보일지 시각화하는 데 도움이 되도록 메타 필드 바로 아래에 페이지의 미리보기가 렌더링됩니다. 이 미리보기는 메타데이터를 편집할 때 실시간으로 업데이트됩니다. 제목 및 설명 필드의 문자 카운터와 같이 효과적인 메타를 작성하는 데 도움이 되는 시각적 표시기도 있습니다. `og:title`이나 `json-ld`와 같이 애플리케이션에서 요구하는 대로 `meta` 필드 그룹에 자체 커스텀 필드를 주입할 수도 있습니다. Yoast SEO와 같은 것을 사용해 본 적이 있다면 이 플러그인이 매우 친숙하게 느껴질 수 있습니다.

## 핵심 기능

- SEO가 활성화된 모든 컬렉션 또는 글로벌에 `meta` 필드 그룹을 추가
- 메타데이터를 자동 생성하는 커스텀 함수를 정의할 수 있음
- 콘텐츠 편집자가 효과적인 메타를 작성할 수 있도록 힌트와 표시기를 표시
- 검색 엔진이 표시할 수 있는 내용의 스니펫을 렌더링
- `og:title` 또는 `json-ld`와 같은 커스텀 필드를 정의할 수 있도록 확장 가능
- 곧 동적 변수 주입을 지원할 예정

## 설치

[pnpm](https://pnpm.io), [npm](https://npmjs.com), [Yarn](https://yarnpkg.com)과 같은 JavaScript 패키지 관리자를 사용하여 플러그인을 설치하세요:

```bash
  pnpm add @payloadcms/plugin-seo
```

## 기본 사용법

[Payload Config](../configuration/overview)의 `plugins` 배열에서 [옵션](#options)과 함께 플러그인을 호출하세요:

```ts
import { buildConfig } from 'payload';
import { seoPlugin } from '@payloadcms/plugin-seo';

const config = buildConfig({
  collections: [
    {
      slug: 'pages',
      fields: []
    },
    {
      slug: 'media',
      upload: {
        staticDir: // 정적 디렉토리 경로,
      },
      fields: []
    }
  ],
  plugins: [
    seoPlugin({
      collections: [
        'pages',
      ],
      uploadsCollection: 'media',
      generateTitle: ({ doc }) => `Website.com — ${doc.title}`,
      generateDescription: ({ doc }) => doc.excerpt
    })
  ]
});

export default config;
```

### 옵션

##### `collections`

SEO를 활성화할 컬렉션 슬러그의 배열. 활성화된 컬렉션은 제목, 설명 및 이미지 하위 필드의 객체인 `meta` 필드를 받습니다.

##### `globals`

SEO를 활성화할 글로벌 슬러그의 배열. 활성화된 글로벌은 제목, 설명 및 이미지 하위 필드의 객체인 `meta` 필드를 받습니다.

##### `fields`

객체를 통해 기본 필드를 받아 필드 배열을 반환할 것으로 예상되는 함수. 이를 사용하여 기존 필드를 수정하거나 새 필드를 추가할 수 있습니다.

```ts
// payload.config.ts
{
  // ...
  seoPlugin({
    fields: ({ defaultFields }) => [
      ...defaultFields,
      {
        name: 'customField',
        type: 'text',
      },
    ],
  })
}
```

##### `uploadsCollection`

애플리케이션의 업로드 활성화 컬렉션 슬러그로 `uploadsCollection`을 설정하세요. 이는 `meta` 필드 그룹에 `image` 필드를 제공하는 데 사용됩니다.

##### `tabbedUI`

`tabbedUI` 속성이 `true`이면 Payload의 [탭 필드](../fields/tabs)를 사용하여 구성에 `SEO` 탭을 추가합니다. 컬렉션이 아직 탭 활성화되지 않은 경우, 즉 구성의 첫 번째 필드가 `tabs` 유형이 아닌 경우 `Content`라는 탭이 생성됩니다. 기본값은 `false`입니다.
구성에서 플러그인이나 필드의 순서는 플러그인이 기존 필드와 탭을 똑똑하게 병합할 수 있는지 여부에 영향을 줄 수 있습니다. 복잡한 구조가 있는 경우 이 구성 옵션에 의존하는 대신 [직접 필드를 사용](#direct-use-of-fields)하는 것을 권장합니다.

<Banner type="info">
  `tabbedUI`와 함께 최상위 또는 사이드바 필드를 계속 사용하려면
  기본 `Content` 탭이 자동으로 생성되지 않도록 해야 합니다(위의 참고 사항 참조).
  대신 구성의 첫 번째 필드를 `tabs` 유형으로 정의하고 다른 모든 필드를
  이와 인접하게 배치해야 합니다.
</Banner>

##### `generateTitle`

문서의 콘텐츠를 포함하여 모든 메타 제목을 반환할 수 있는 함수.

```ts
// payload.config.ts
{
  // ...
  seoPlugin({
    generateTitle: ({ doc }) => `Website.com — ${doc?.title}`,
  })
}
```

모든 "생성" 함수는 다음 인수를 받습니다:

| 인수                       | 설명                                                                  |
| -------------------------- | --------------------------------------------------------------------- |
| **`collectionConfig`**     | 컬렉션의 구성.                                                        |
| **`collectionSlug`**       | 컬렉션의 슬러그.                                                      |
| **`doc`**                  | 현재 문서의 데이터.                                                   |
| **`docPermissions`**       | 문서의 권한.                                                          |
| **`globalConfig`**         | 글로벌의 구성.                                                        |
| **`globalSlug`**           | 글로벌의 슬러그.                                                      |
| **`hasPublishPermission`** | 사용자가 문서를 게시할 권한이 있는지 여부.                            |
| **`hasSavePermission`**    | 사용자가 문서를 저장할 권한이 있는지 여부.                            |
| **`id`**                   | 문서의 ID.                                                            |
| **`initialData`**          | 문서의 초기 데이터.                                                   |
| **`initialState`**         | 문서의 초기 상태.                                                     |
| **`locale`**               | 문서의 로케일.                                                        |
| **`preferencesKey`**       | 문서의 기본 설정 키.                                                  |
| **`publishedDoc`**         | 게시된 문서.                                                          |
| **`req`**                  | `user`, `payload`, `i18n` 등을 포함하는 Payload 요청 객체.           |
| **`title`**                | 문서의 제목.                                                          |
| **`versionsCount`**        | 문서의 버전 수.                                                       |

##### `generateDescription`

문서의 콘텐츠를 포함하여 모든 메타 설명을 반환할 수 있는 함수.

```ts
// payload.config.ts
{
  // ...
  seoPlugin({
    generateDescription: ({ doc }) => doc?.excerpt,
  })
}
```

전체 인수 목록은 [`generateTitle`](#generatetitle) 함수를 참조하세요.

##### `generateImage`

문서의 콘텐츠를 포함하여 모든 메타 이미지를 반환할 수 있는 함수.

```ts
// payload.config.ts
{
  // ...
  seoPlugin({
    generateImage: ({ doc }) => doc?.featuredImage,
  })
}
```

전체 인수 목록은 [`generateTitle`](#generatetitle) 함수를 참조하세요.

##### `generateURL`

페이지의 실제 URL을 표시하기 위해 검색 미리보기 컴포넌트에서 호출되는 함수.

```ts
// payload.config.ts
{
  // ...
  seoPlugin({
    generateURL: ({ doc, collectionSlug }) =>
      `https://yoursite.com/${collectionSlug}/${doc?.slug}`,
  })
}
```

전체 인수 목록은 [`generateTitle`](#generatetitle) 함수를 참조하세요.

#### `interfaceName`

TypeScript 및 GraphQL용으로 생성되는 메타 그룹 인터페이스 이름을 변경합니다.

```ts
// payload.config.ts
{
  // ...
  seoPlugin({
    interfaceName: 'customInterfaceNameSEO',
  })
}
```

## 필드의 직접 사용

플러그인에서 필드를 직접 가져와서 필요에 따라 어디든 포함시킬 수 있는 옵션이 있습니다.

<Banner type="info">
  생성 함수를 구성하려면 여전히 Payload Config에서 플러그인을 구성해야 합니다.
  이러한 필드는 직접 가져와서 사용되므로 플러그인 구성에 액세스할 수 없어
  동일한 방식으로 작동하려면 추가 인수가 필요할 수 있습니다.
</Banner>

```ts
import {
  MetaDescriptionField,
  MetaImageField,
  MetaTitleField,
  OverviewField,
  PreviewField,
} from '@payloadcms/plugin-seo/fields'

// 필드로 사용됨
MetaImageField({
  // 업로드 컬렉션 슬러그
  relationTo: 'media',

  // `generateImage` 함수가 구성된 경우
  hasGenerateFn: true,
})

MetaDescriptionField({
  // `generateDescription` 함수가 구성된 경우
  hasGenerateFn: true,
})

MetaTitleField({
  // `generateTitle` 함수가 구성된 경우
  hasGenerateFn: true,
})

PreviewField({
  // `generateUrl` 함수가 구성된 경우
  hasGenerateFn: true,

  // 데이터의 대상 필드와 일치할 필드 경로
  titlePath: 'meta.title',
  descriptionPath: 'meta.description',
})

OverviewField({
  // 데이터의 대상 필드와 일치할 필드 경로
  titlePath: 'meta.title',
  descriptionPath: 'meta.description',
  imagePath: 'meta.image',
})
```

<Banner type="info">
  팁: 필드의 minLength 및 maxLength props를 변경하여 길이 규칙을 재정의할 수 있습니다.
  OverviewField의 경우 `titleOverrides` 및 `descriptionOverrides`를 사용하여
  길이 규칙을 재정의할 수 있습니다.
</Banner>

## TypeScript

모든 타입은 직접 가져올 수 있습니다:

```ts
import type {
  PluginConfig,
  GenerateTitle,
  GenerateDescription
  GenerateURL
} from '@payloadcms/plugin-seo/types';
```

그런 다음 생성된 Payload 타입에서 컬렉션을 생성 타입에 전달할 수 있습니다. 예를 들어:

```ts
import type { Page } from './payload-types.ts'

import type { GenerateTitle } from '@payloadcms/plugin-seo/types'

const generateTitle: GenerateTitle<Page> = async ({ doc, locale }) => {
  return `Website.com — ${doc?.title}`
}
```
