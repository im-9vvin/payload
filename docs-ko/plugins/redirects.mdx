![https://www.npmjs.com/package/@payloadcms/plugin-redirects](https://img.shields.io/npm/v/@payloadcms/plugin-redirects)

이 플러그인을 사용하면 [관리자 패널](../admin/overview) 내에서 애플리케이션의 리디렉트를 쉽게 관리할 수 있습니다. 한 URL에서 다른 URL로의 리디렉트를 지정할 수 있는 `redirects` 컬렉션을 구성에 추가하는 방식으로 작동합니다. 프론트엔드 애플리케이션은 이 데이터를 사용하여 적절한 HTTP 상태 코드를 사용해 사용자를 올바른 페이지로 자동 리디렉트할 수 있습니다. 이는 플랫폼을 재구축하거나 URL 구조를 변경할 때 SEO, 인덱싱 및 검색 엔진 순위에 유용합니다.

예를 들어, `/about`에 페이지가 있고 이를 `/about-us`로 변경하려는 경우 이전 페이지에서 새 페이지로의 리디렉트를 생성한 다음 이 데이터를 사용하여 프론트엔드 애플리케이션에 HTTP 리디렉트를 작성할 수 있습니다. 이렇게 하면 검색 엔진이 요청 수준에서 변경 사항을 알림받기 때문에 사용자가 페널티 없이 올바른 페이지로 리디렉트됩니다. 이는 모든 프론트엔드 프레임워크에 대해 관리되는 리디렉트를 통합할 수 있는 매우 가벼운 플러그인입니다.

## 핵심 기능

- 구성에 `redirects` 컬렉션을 추가하여:
  - `from` 및 `to` 필드 포함
  - `to`가 문서 참조가 될 수 있도록 허용

## 설치

[pnpm](https://pnpm.io), [npm](https://npmjs.com), [Yarn](https://yarnpkg.com)과 같은 JavaScript 패키지 관리자를 사용하여 플러그인을 설치하세요:

```bash
  pnpm add @payloadcms/plugin-redirects
```

## 기본 사용법

[Payload Config](https://payloadcms.com/docs/configuration/overview)의 `plugins` 배열에서 [옵션](#options)과 함께 플러그인을 호출하세요:

```ts
import { buildConfig } from 'payload'
import { redirectsPlugin } from '@payloadcms/plugin-redirects'

const config = buildConfig({
  collections: [
    {
      slug: 'pages',
      fields: [],
    },
  ],
  plugins: [
    redirectsPlugin({
      collections: ['pages'],
    }),
  ],
})

export default config
```

### 옵션

| 옵션                        | 타입       | 설명                                                                                               |
| --------------------------- | ---------- | -------------------------------------------------------------------------------------------------- |
| `collections`               | `string[]` | 각 리디렉트의 `to` 필드에 채울 컬렉션 슬러그의 배열.                                              |
| `overrides`                 | `object`   | `redirects` 컬렉션의 모든 것을 재정의할 수 있는 부분 컬렉션 구성.                                  |
| `redirectTypes`             | `string[]` | 지원할 리디렉트 유형에 대한 옵션을 제공하려는 경우 리디렉트 배열을 제공.                           |
| `redirectTypeFieldOverride` | `Field`    | 위에서 활성화된 경우 리디렉트 유형 필드를 재정의할 수 있는 부분 필드 구성.                         |

overrides의 필드는 기본 필드를 받아서 필드 배열을 반환하는 함수를 사용합니다. 이를 통해 컬렉션에 필드를 추가할 수 있습니다.

```ts
redirectsPlugin({
  collections: ['pages'],
  overrides: {
    fields: ({ defaultFields }) => {
      return [
        ...defaultFields,
        {
          type: 'text',
          name: 'customField',
        },
      ]
    },
  },
  redirectTypes: ['301', '302'],
  redirectTypeFieldOverride: {
    label: 'Redirect Type (Overridden)',
  },
})
```

## TypeScript

모든 타입은 직접 가져올 수 있습니다:

```ts
import { PluginConfig } from '@payloadcms/plugin-redirects/types'
```

[템플릿 디렉토리](https://github.com/payloadcms/payload/tree/main/templates)에는 이 플러그인을 사용하는 공식 [웹사이트 템플릿](https://github.com/payloadcms/payload/tree/main/templates/website) 및 [전자상거래 템플릿](https://github.com/payloadcms/payload/tree/main/templates/ecommerce)도 포함되어 있습니다.
