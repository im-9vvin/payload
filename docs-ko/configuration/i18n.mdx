[관리자 패널](../admin/overview)은 [30개 이상의 언어](https://github.com/payloadcms/payload/tree/main/packages/translations)로 번역되어 있습니다. I18n을 사용하면 편집자가 원하는 언어로 인터페이스를 탐색하고 API 오류 메시지를 읽을 수 있습니다. 이는 [지역화](./localization)와 유사하지만, 데이터 자체의 번역을 관리하는 것이 아니라 애플리케이션의 인터페이스 번역을 관리합니다.

기본적으로 Payload는 영어로 미리 설치되지만, 다른 언어를 애플리케이션에 쉽게 로드할 수 있습니다. 언어는 요청을 기반으로 자동으로 감지됩니다. 언어가 감지되지 않거나 사용자의 언어가 아직 애플리케이션에서 지원되지 않는 경우 영어가 선택됩니다.

프로젝트에 I18n을 추가하려면 먼저 `@payloadcms/translations` 패키지를 설치해야 합니다:

```bash
pnpm install @payloadcms/translations
```

설치가 완료되면 [Payload 설정](./overview)에서 `i18n` 키를 사용하여 구성할 수 있습니다:

```ts
import { buildConfig } from 'payload'

export default buildConfig({
  // ...
  i18n: {
    // highlight-line
    // ...
  },
})
```

<Banner type="success">
  **참고:** Payload가 아직 지원하지 않는 언어가 있다면, [코드 기여](https://github.com/payloadcms/payload/blob/main/CONTRIBUTING.md)를 환영합니다.
</Banner>

## 설정 옵션

Payload가 기본적으로 제공하는 i18n 설정을 쉽게 사용자 정의하고 재정의할 수 있습니다. Payload는 사용자 정의 옵션을 사용하고 자체 설정과 병합합니다.

```ts
import { buildConfig } from 'payload'

export default buildConfig({
  // ...
  // highlight-start
  i18n: {
    fallbackLanguage: 'en', // 기본값
  },
  // highlight-end
})
```

다음 옵션들을 사용할 수 있습니다:

| 옵션                 | 설명                                                                                                              |
| -------------------- | ------------------------------------------------------------------------------------------------------------------ |
| `fallbackLanguage`   | 사용자의 선호 언어가 지원되지 않는 경우 대체할 언어입니다. 기본값은 `'en'`입니다.                                |
| `translations`       | 번역이 포함된 객체입니다. 키는 언어 코드이고 값은 번역입니다.                                                    |
| `supportedLanguages` | 지원되는 언어가 포함된 객체입니다. 키는 언어 코드이고 값은 번역입니다.                                           |

## 언어 추가

새 언어에 대한 번역을 제공하여 Payload 앱에 새 언어를 쉽게 추가할 수 있습니다. Payload는 `@payloadcms/translations`에서 가져올 수 있는 여러 내장 번역을 유지 관리하지만, 모든 언어를 지원하기 위해 자체 [사용자 정의 번역](#custom-translations)을 제공할 수도 있습니다.

새 언어를 추가하려면 [Payload 설정](./overview)에서 `i18n.supportedLanguages` 키를 사용하세요:

```ts
import { buildConfig } from 'payload'
import { en } from '@payloadcms/translations/languages/en'
import { de } from '@payloadcms/translations/languages/de'

export default buildConfig({
  // ...
  // highlight-start
  i18n: {
    supportedLanguages: { en, de },
  },
  // highlight-end
})
```

<Banner type="warning">
  **팁:** 프로젝트의 번들된 JavaScript를 최소한으로 유지하기 위해 필요한 언어만 지원하는 것이 좋습니다.
</Banner>

### 사용자 정의 번역

기존 언어를 확장하거나 새 언어를 완전히 추가하여 Payload의 내장 번역을 사용자 정의할 수 있습니다. 기존 언어에 새 번역 문자열을 주입하거나 완전히 새로운 언어 키를 제공하여 수행할 수 있습니다.

사용자 정의 번역을 추가하려면 [Payload 설정](./overview)에서 `i18n.translations` 키를 사용하세요:

```ts
import { buildConfig } from 'payload'

export default buildConfig({
  //...
  i18n: {
    // highlight-start
    translations: {
      en: {
        custom: {
          // namespace can be anything you want
          key1: 'Translation with {{variable}}', // translation
        },
        // override existing translation keys
        general: {
          dashboard: 'Home',
        },
      },
    },
    // highlight-end
  },
  //...
})
```

### 프로젝트 번역

Payload의 내장 기능은 완전히 번역되어 제공되지만, 자체 프로젝트의 일부도 번역하고 싶을 수 있습니다. [컬렉션](./collections) 및 [글로벌](./globals)과 같은 곳에서 레이블과 그룹, 필드 레이블, 설명 또는 입력 플레이스홀더 텍스트를 번역할 수 있습니다.

이를 위해 해당하는 곳에서 언어 코드를 키로 하여 번역을 제공하세요:

```ts
import type { CollectionConfig } from 'payload'

export const Articles: CollectionConfig = {
  slug: 'articles',
  labels: {
    singular: {
      // highlight-start
      en: 'Article',
      es: 'Artículo',
      // highlight-end
    },
    plural: {
      // highlight-start
      en: 'Articles',
      es: 'Artículos',
      // highlight-end
    },
  },
  admin: {
    group: {
      // highlight-start
      en: 'Content',
      es: 'Contenido',
      // highlight-end
    },
  },
  fields: [
    {
      name: 'title',
      type: 'text',
      label: {
        // highlight-start
        en: 'Title',
        es: 'Título',
        // highlight-end
      },
      admin: {
        placeholder: {
          // highlight-start
          en: 'Enter title',
          es: 'Introduce el título',
          // highlight-end
        },
      },
    },
  ],
}
```

## 언어 변경

사용자는 계정 설정에서 또는 [사용자 환경설정](../admin/preferences)을 조작하여 선호하는 언어를 변경할 수 있습니다.

## Node.js

Payload의 백엔드는 요청이 처리되기 전에 들어오는 요청의 언어를 설정합니다. 이를 통해 백엔드 유효성 검사가 사용자의 언어로 오류 메시지를 반환하거나 시스템에서 생성된 이메일이 올바른 번역을 사용하여 전송될 수 있습니다. `accept-language` 헤더와 함께 HTTP 요청을 만들면 Payload가 해당 언어를 사용합니다.

Payload 앱에서 `req` 객체에 액세스할 수 있는 모든 곳에서 `req.i18n`에 할당된 Payload의 광범위한 국제화 기능에 액세스할 수 있습니다. 텍스트 번역에 액세스하려면 `req.t('namespace:key')`를 사용할 수 있습니다.

## TypeScript

프로젝트에서 [사용자 정의 번역](#custom-translations)을 사용하려면 번역에 대한 타입을 제공해야 합니다.

여기서는 공유 가능한 번역 객체를 만듭니다. 사용자 정의 컴포넌트와 Payload 설정 모두에서 이를 가져올 것입니다.

이 예제에서는 영어를 확장하는 방법을 보여주지만, 원하는 모든 언어에 대해 동일하게 수행할 수 있습니다.

```ts
// <rootDir>/custom-translations.ts

import { enTranslations } from '@payloadcms/translations/languages/en'
import type { NestedKeysStripped } from '@payloadcms/translations'

export const customTranslations = {
  en: {
    general: {
      myCustomKey: 'My custom english translation',
    },
    fields: {
      addLabel: 'Add!',
    },
  },
}

export type CustomTranslationsObject = typeof customTranslations.en &
  typeof enTranslations
export type CustomTranslationsKeys =
  NestedKeysStripped<CustomTranslationsObject>
```

공유 번역 객체를 Payload 설정으로 가져와서 사용할 수 있도록 합니다:

```ts
// <rootDir>/payload.config.ts

import { buildConfig } from 'payload'

import { customTranslations } from './custom-translations'

export default buildConfig({
  //...
  i18n: {
    translations: customTranslations,
  },
  //...
})
```

[사용자 정의 컴포넌트](../custom-components/overview)에서 사용할 공유 번역 타입을 가져옵니다:

```ts
// <rootDir>/components/MyComponent.tsx

'use client'
import type React from 'react'
import { useTranslation } from '@payloadcms/ui'

import type {
  CustomTranslationsObject,
  CustomTranslationsKeys,
} from '../custom-translations'

export const MyComponent: React.FC = () => {
  const { i18n, t } = useTranslation<
    CustomTranslationsObject,
    CustomTranslationsKeys
  >() // These generics merge your custom translations with the default client translations

  return t('general:myCustomKey')
}
```

또한 Payload는 레이블과 같은 다양한 곳에서 `t` 함수를 노출합니다. 다음과 같이 타입을 지정할 수 있습니다:

```ts
// <rootDir>/fields/myField.ts

import type {
  DefaultTranslationKeys,
  TFunction,
} from '@payloadcms/translations'
import type { Field } from 'payload'

import { CustomTranslationsKeys } from '../custom-translations'

const field: Field = {
  name: 'myField',
  type: 'text',
  label: ({ t: defaultT }) => {
    const t = defaultT as TFunction<CustomTranslationsKeys>
    return t('fields:addLabel')
  },
}
```
