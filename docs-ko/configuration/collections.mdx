컬렉션은 문서라고 하는 레코드들의 그룹으로, 모두 공통 스키마를 공유합니다. 애플리케이션에 필요한 만큼의 컬렉션을 정의할 수 있습니다. 컬렉션의 각 문서는 정의한 [필드](../fields/overview)를 기반으로 [데이터베이스](../database/overview)에 저장되며, 문서를 관리하는 데 사용되는 [로컬 API](../local-api/overview), [REST API](../rest-api/overview), 및 [GraphQL API](../graphql/overview)를 자동으로 생성합니다.

컬렉션은 Payload에서 [인증](../authentication/overview)을 구현하는 데도 사용됩니다. `auth` 옵션을 사용하여 컬렉션을 정의하면, 해당 컬렉션은 사용자 인증을 지원하는 추가 작업을 받습니다.

컬렉션은 사용자, 제품, 페이지, 게시물 및 관리하고자 하는 기타 유형의 콘텐츠와 같이 애플리케이션에서 반복되는 데이터를 구조화하는 주요 방법입니다. 각 컬렉션은 고유한 [접근 제어](../access-control/overview), [훅](../hooks/overview), [관리자 옵션](#admin-options) 등을 가질 수 있습니다.

컬렉션 구성을 정의하려면 [Payload 구성](./overview)에서 `collection` 속성을 사용하세요:

```ts
import { buildConfig } from 'payload'

export default buildConfig({
  // ...
  collections: [
    // highlight-line
    // 컬렉션이 여기에 들어갑니다
  ],
})
```

<Banner type="success">
  **팁:** 컬렉션이 단일 문서만 포함하도록 의도된 경우,
  대신 [글로벌](./globals) 사용을 고려하세요.
</Banner>

## 구성 옵션

컬렉션을 별도의 파일에 작성한 다음 주요 [Payload 구성](./overview)으로 가져오는 것이 모범 사례인 경우가 많습니다.

간단한 컬렉션 구성이 어떻게 생겼는지는 다음과 같습니다:

```ts
import type { CollectionConfig } from 'payload'

export const Posts: CollectionConfig = {
  slug: 'posts',
  fields: [
    {
      name: 'title',
      type: 'text',
    },
  ],
}
```

<Banner type="success">
  **알림:** 더 복잡한 예시는 Payload 저장소의
  [템플릿](https://github.com/payloadcms/payload/tree/main/templates) 및
  [예시](https://github.com/payloadcms/payload/tree/main/examples)
  디렉터리를 참조하세요.
</Banner>

다음 옵션을 사용할 수 있습니다:

| 옵션                 | 설명                                                                                                                                                                                                                             |
| -------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `admin`              | 관리 패널에 대한 구성 옵션입니다. [자세한 내용](#admin-options).                                                                                                                                                                |
| `access`             | 이 컬렉션의 문서에 대해 누가 무엇을 할 수 있는지 정확히 정의하는 접근 제어 함수를 제공합니다. [자세한 내용](../access-control/collections).                                                                                     |
| `auth`               | 이 컬렉션에 인증 기능을 추가하려면 옵션을 지정하세요. [자세한 내용](../authentication/overview).                                                                                                                                |
| `custom`             | 사용자 정의 데이터 추가를 위한 확장 지점입니다(예: 플러그인용).                                                                                                                                                                 |
| `disableDuplicate`   | true인 경우, 이 컬렉션 내의 문서를 편집하는 동안 "복제" 버튼을 표시하지 않고 모든 API에서 `duplicate`를 방지합니다.                                                                                                             |
| `defaultSort`        | 컬렉션 목록 뷰에서 기본으로 정렬할 최상위 필드를 전달합니다. 내림차순으로 정렬하려면 필드 이름 앞에 마이너스 기호("-")를 붙이세요. 문자열 배열을 사용하여 여러 필드를 지정할 수 있습니다.                                       |
| `dbName`             | 데이터베이스 어댑터에 따라 사용자 정의 테이블 또는 컬렉션 이름입니다. 정의되지 않은 경우 slug에서 자동 생성됩니다.                                                                                                               |
| `endpoints`          | REST API에 사용자 정의 경로를 추가합니다. 경로를 비활성화하려면 `false`로 설정하세요. [자세한 내용](../rest-api/overview#custom-endpoints).                                                                                     |
| `fields` \*          | 이 컬렉션 내에 저장된 데이터의 구조와 기능을 결정하는 필드 유형의 배열입니다. [자세한 내용](../fields/overview).                                                                                                                |
| `graphQL`            | 이 컬렉션에 대한 GraphQL 관련 속성을 관리합니다. [자세한 내용](#graphql)                                                                                                                                                        |
| `hooks`              | 훅의 진입점입니다. [자세한 내용](../hooks/overview#collection-hooks).                                                                                                                                                          |
| `orderable`          | true인 경우, 컬렉션에 대한 사용자 정의 순서를 활성화하고, 문서를 드래그 앤 드롭으로 재정렬할 수 있습니다. 효율적인 재정렬을 위해 [분수 인덱싱](https://observablehq.com/@dgreensp/implementing-fractional-indexing)을 사용합니다. |
| `labels`             | Payload 전체에서 이 컬렉션을 식별하는 데 사용되는 단수 및 복수 라벨입니다. 정의되지 않은 경우 slug에서 자동 생성됩니다.                                                                                                          |
| `enableQueryPresets` | 이 컬렉션에 대한 쿼리 프리셋을 활성화합니다. [자세한 내용](../query-presets/overview).                                                                                                                                          |
| `lockDocuments`      | 문서 잠금을 활성화 또는 비활성화합니다. 기본적으로 문서 잠금이 활성화됩니다. 구성하려면 객체로 설정하거나, 잠금을 비활성화하려면 `false`로 설정하세요. [자세한 내용](../admin/locked-documents).                              |
| `slug` \*            | 이 컬렉션의 식별자 역할을 할 고유하고 URL 친화적인 문자열입니다.                                                                                                                                                                |
| `timestamps`         | 문서의 자동 생성된 `createdAt` 및 `updatedAt` 타임스탬프를 비활성화하려면 false로 설정하세요.                                                                                                                                   |
| `trash`              | 이 컬렉션에 대한 소프트 삭제를 활성화하는 불린값입니다. 기본값은 `false`입니다. [자세한 내용](../trash/overview).                                                                                                               |
| `typescript`         | 스키마 생성에 사용되는 텍스트로 `interface` 속성을 가진 객체입니다. 정의되지 않은 경우 slug에서 자동 생성됩니다.                                                                                                                |
| `upload`             | 이 컬렉션이 파일 업로드를 지원하도록 하려면 옵션을 지정하세요. 자세한 내용은 [업로드](../upload/overview) 문서를 참조하세요.                                                                                                   |
| `versions`           | 기본 옵션을 활성화하려면 true로 설정하거나 객체 속성으로 구성하세요. [자세한 내용](../versions/overview#collection-config).                                                                                                  |
| `defaultPopulate`    | 이 컬렉션이 다른 문서에서 채워질 때 선택할 필드를 지정합니다. [자세한 내용](../queries/select#defaultpopulate-collection-config-property).                                                                                   |
| `indexes`            | 이 컬렉션에 대한 복합 인덱스를 정의합니다. 2개 이상의 필드로 동시에 쿼리/정렬 속도를 높이거나 여러 필드 간의 고유성을 보장하는 데 사용할 수 있습니다.                                                                          |
| `forceSelect`        | `select` 쿼리에 관계없이 항상 선택되어야 하는 필드를 지정합니다. 접근 제어/훅에 필드가 존재하는 것이 유용할 수 있습니다                                                                                                        |
| `disableBulkEdit`    | 관리 패널과 REST API에서 컬렉션의 대량 편집 작업을 비활성화합니다                                                                                                                                                               |

_\* 별표는 필수 속성을 나타냅니다._

### 필드

필드는 컬렉션 내 문서의 스키마를 정의합니다. 자세히 알아보려면 [필드](../fields/overview) 문서로 이동하세요.

### 접근 제어

[컬렉션 접근 제어](../access-control/overview)는 사용자가 컬렉션 내의 주어진 문서로 무엇을 할 수 있고 할 수 없는지를 결정합니다. 자세히 알아보려면 [접근 제어](../access-control/overview) 문서로 이동하세요.

## 관리자 옵션

[관리 패널](../admin/overview) 내에서 컬렉션의 동작은 애플리케이션의 필요에 맞게 완전히 사용자 정의할 수 있습니다. 여기에는 탐색 링크를 그룹화하거나 숨기기, [사용자 정의 컴포넌트](../custom-components/overview) 추가, 목록 뷰에 표시할 필드 선택 등이 포함됩니다.

컬렉션에 대한 관리자 옵션을 구성하려면 컬렉션 구성에서 `admin` 속성을 사용하세요:

```ts
import type { CollectionConfig } from 'payload'

export const MyCollection: CollectionConfig = {
  // ...
  admin: {
    // highlight-line
    // ...
  },
}
```

다음 옵션을 사용할 수 있습니다:

| 옵션                         | 설명                                                                                                                                                                                                                 |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `group`                      | 관리자 탐색에서 컬렉션 및 글로벌 링크를 그룹화하는 데 사용되는 텍스트 또는 지역화 객체입니다. 경로에 액세스할 수 있게 하면서 탐색에서 링크를 숨기려면 `false`로 설정하세요.                                           |
| `hidden`                     | 이 컬렉션을 탐색 및 관리자 라우팅에서 제외하려면 true로 설정하거나 현재 사용자와 함께 호출되어 true를 반환하는 함수로 설정하세요.                                                                                      |
| `hooks`                      | 이 컬렉션에 대한 관리자별 훅입니다. [자세한 내용](../hooks/collections).                                                                                                                                             |
| `useAsTitle`                 | 관리 패널 전체에서 문서 제목으로 사용할 최상위 필드를 지정합니다. 필드가 정의되지 않은 경우, 문서의 ID가 제목으로 사용됩니다.                                                                                        |
| `description`                | 편집자에게 더 많은 정보를 제공하기 위해 목록 뷰의 컬렉션 라벨 아래에 표시할 텍스트입니다. 또는 React 컴포넌트를 렌더링하기 위해 `admin.components.Description`을 사용할 수 있습니다. [자세한 내용](#custom-components). |
| `defaultColumns`             | 이 컬렉션의 목록 뷰에서 기본으로 표시할 열에 해당하는 필드 이름의 배열입니다.                                                                                                                                        |
| `disableCopyToLocale`        | 이 컬렉션 내의 문서를 편집하는 동안 "로케일로 복사" 버튼을 비활성화합니다. 지역화가 활성화된 경우에만 적용됩니다.                                                                                                   |
| `groupBy`                    | 베타. 목록 뷰에서 필드별 그룹화를 활성화합니다.                                                                                                                                                                      |
| `hideAPIURL`                 | 이 컬렉션 내의 문서를 편집하는 동안 "API URL" 메타 필드를 숨깁니다.                                                                                                                                                  |
| `enableRichTextLink`         | [리치 텍스트](../fields/rich-text) 필드는 사용자가 리치 텍스트 내에서 관련 문서를 자동으로 참조할 수 있는 `Link` 요소를 제공합니다. 기본적으로 `true`로 설정됩니다.                                                |
| `enableRichTextRelationship` | [리치 텍스트](../fields/rich-text) 필드는 사용자가 리치 텍스트 내에서 관련 문서를 자동으로 참조할 수 있는 `Relationship` 요소를 제공합니다. 기본적으로 `true`로 설정됩니다.                                        |
| `folders`                    | 주어진 컬렉션에 대해 폴더를 활성화하는 불린값입니다. 기본값은 `false`입니다. [자세한 내용](../folders/overview).                                                                                                     |
| `meta`                       | 관리 패널 내에서 이 컬렉션에 적용할 페이지 메타데이터 재정의입니다. [자세한 내용](../admin/metadata).                                                                                                                |
| `preview`                    | 앱을 가리킬 수 있는 관리 패널 내에서 미리보기 URL을 생성하는 함수입니다. [자세한 내용](../admin/preview).                                                                                                            |
| `livePreview`                | 프론트엔드 애플리케이션의 즉각적인 시각적 피드백을 위한 실시간 편집을 활성화합니다. [자세한 내용](../live-preview/overview).                                                                                         |
| `components`                 | 이 컬렉션 내에서 사용할 자신의 React 컴포넌트로 교체합니다. [자세한 내용](#custom-components).                                                                                                                      |
| `listSearchableFields`       | 목록 검색 뷰에서 검색해야 할 필드를 지정합니다. [자세한 내용](#list-searchable-fields).                                                                                                                             |
| `pagination`                 | 이 컬렉션에 대한 페이지네이션별 옵션을 설정합니다. [자세한 내용](#pagination).                                                                                                                                       |
| `baseListFilter`             | 이 컬렉션의 목록 뷰에 대한 기본 베이스 필터를 정의할 수 있으며, 사용자가 수행하는 모든 필터에 병합됩니다.                                                                                                           |

<Banner type="warning">
  **참고:** `useAsTitle`을 관계 또는 조인 필드로 설정하면, 관련 문서의 ID만을
  제목으로 사용합니다. 관련 문서에서 특정 필드(예: title)를 대신 표시하려면,
  원하는 데이터를 추출하는 가상 필드를 생성하고 `useAsTitle`을 해당 가상 필드로
  설정하세요.
</Banner>

### 사용자 정의 컴포넌트

컬렉션은 [관리 패널](../admin/overview) 내에서 컬렉션별 UI에만 적용되는 고유한 [사용자 정의 컴포넌트](../custom-components/overview)를 설정할 수 있습니다. 여기에는 저장 버튼과 같은 요소나 편집 뷰와 같은 전체 레이아웃이 포함됩니다.

컬렉션 컴포넌트를 재정의하려면 컬렉션 구성에서 `admin.components` 속성을 사용하세요:

```ts
import type { CollectionConfig } from 'payload'

export const MyCollection: CollectionConfig = {
  // ...
  admin: {
    components: {
      // highlight-line
      // ...
    },
  },
}
```

다음 옵션을 사용할 수 있습니다:

| 옵션              | 설명                                                                                                                                                                |
| ----------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `afterList`       | 내장 목록 뷰 _이후_에 삽입할 컴포넌트 배열입니다. [자세한 내용](../custom-components/list-view#afterlist).                                                         |
| `afterListTable`  | 내장 목록 뷰의 테이블 _이후_에 삽입할 컴포넌트 배열입니다. [자세한 내용](../custom-components/list-view#afterlisttable).                                           |
| `beforeList`      | 내장 목록 뷰 _이전_에 삽입할 컴포넌트 배열입니다. [자세한 내용](../custom-components/list-view#beforelist).                                                        |
| `beforeListTable` | 내장 목록 뷰의 테이블 _이전_에 삽입할 컴포넌트 배열입니다. [자세한 내용](../custom-components/list-view#beforelisttable).                                          |
| `listMenuItems`   | 목록 컨트롤 옆의 메뉴 내에서 렌더링할 컴포넌트 배열입니다(열 및 필터 옵션 이후)                                                                                    |
| `Description`     | 목록 뷰에서 컬렉션 라벨 아래에 렌더링할 컴포넌트입니다. `admin.description` 속성의 대안입니다. [자세한 내용](../custom-components/list-view#description).         |
| `edit`            | 편집 뷰 내의 특정 컴포넌트를 재정의합니다. [자세한 내용](#edit-view-options).                                                                                      |
| `views`           | 관리 패널 내에서 뷰를 재정의하거나 새로 생성합니다. [자세한 내용](../custom-components/custom-views).                                                              |

#### 편집 뷰 옵션

```ts
import type { CollectionConfig } from 'payload'

export const MyCollection: CollectionConfig = {
  // ...
  admin: {
    components: {
      edit: {
        // highlight-line
        // ...
      },
    },
  },
}
```

다음 옵션을 사용할 수 있습니다:

| 옵션                     | 설명                                                                                                                                                                                                 |

### 페이지네이션

모든 컬렉션은 정렬 및 필터링 할 수 있는 문서의 페이지네이션된 목록을 표시하는 고유한 목록 뷰를 받습니다. 목록 뷰의 페이지네이션 동작은 컬렉션별로 사용자 정의할 수 있으며, Payload가 제공하는 동일한 [페이지네이션](../queries/pagination) API를 사용합니다.

페이지네이션 옵션을 구성하려면 컬렉션 구성에서 `admin.pagination` 속성을 사용하세요:

```ts
import type { CollectionConfig } from 'payload'

export const Posts: CollectionConfig = {
  // ...
  admin: {
    // highlight-start
    pagination: {
      defaultLimit: 10,
      limits: [10, 20, 50],
    },
    // highlight-end
  },
}
```

다음 옵션을 사용할 수 있습니다:

| 옵션           | 설명                                                                                                       |
| -------------- | ---------------------------------------------------------------------------------------------------------- |
| `defaultLimit` | 사용해야 하는 기본 페이지당 제한을 지정하는 정수입니다. 기본값은 10입니다.                                 |
| `limits`       | 목록 뷰에서 관리자가 선택할 수 있는 페이지당 옵션으로 사용할 정수 배열을 제공합니다.                       |

### 목록 검색 가능 필드

목록 뷰에는 간단한 텍스트 검색을 통해 문서를 빠르게 찾을 수 있는 "검색" 상자가 있습니다. 기본적으로 ID 필드를 검색합니다. 정의된 경우 `admin.useAsTitle` 필드가 사용됩니다. 또는 애플리케이션의 필요에 따라 검색할 필드를 명시적으로 정의할 수 있습니다.

검색해야 할 필드를 정의하려면 컬렉션 구성에서 `admin.listSearchableFields` 속성을 사용하세요:

```ts
import type { CollectionConfig } from 'payload'

export const Posts: CollectionConfig = {
  // ...
  admin: {
    // highlight-start
    listSearchableFields: ['title', 'slug'],
    // highlight-end
  },
}
```

<Banner type="warning">
  **팁:** `listSearchableFields`를 추가하는 경우, 관리자 쿼리의 성능을 유지할 수
  있도록 이러한 각 필드를 인덱싱해야 합니다.
</Banner>

## GraphQL

컬렉션 구성에 `graphQL: false`를 전달하여 이 컬렉션에 대한 GraphQL을 완전히 비활성화할 수 있습니다. 이렇게 하면 GraphQL 스키마에 나타나는 모든 쿼리, 뮤테이션 및 타입이 완전히 비활성화됩니다.

컬렉션의 `graphQL` 속성에 객체를 전달할 수도 있으며, 이를 통해 다음 속성을 정의할 수 있습니다:

| 옵션               | 설명                                                                                     |
| ------------------ | ---------------------------------------------------------------------------------------- |
| `singularName`     | GraphQL 스키마 생성에 사용될 "단수" 이름을 재정의합니다.                                 |
| `pluralName`       | GraphQL 스키마 생성에 사용될 "복수" 이름을 재정의합니다.                                 |
| `disableQueries`   | `true`를 전달하여 이 컬렉션에 해당하는 모든 GraphQL 쿼리를 비활성화합니다.               |
| `disableMutations` | `true`를 전달하여 이 컬렉션에 해당하는 모든 GraphQL 뮤테이션을 비활성화합니다.           |

## TypeScript

Payload에서 타입을 가져와서 컬렉션 구성 작성을 더 쉽고 타입 안전하게 만들 수 있습니다. 컬렉션 구성을 나타내는 두 가지 주요 타입인 `CollectionConfig`와 `SanitizedCollectionConfig`가 있습니다.

`CollectionConfig` 타입은 최소한의 속성만 필수로 표시되는 전체 형태의 원시 컬렉션 구성을 나타냅니다. `SanitizedCollectionConfig` 타입은 완전히 정제된 후의 컬렉션 구성을 나타냅니다. 일반적으로 이는 Payload에서 내부적으로만 사용됩니다.

```ts
import type { CollectionConfig, SanitizedCollectionConfig } from 'payload'
```
