Payload에서 로컬 API 작업은 **기본적으로 접근 제어를 무시합니다**. 이는 현재 사용자가 작업을 수행할 권한이 있는지 확인하지 않고 작업이 실행된다는 의미입니다. 접근 제어가 필요하지 않은 특정 시나리오에서는 유용하지만, 보안상의 이유로 언제 이를 시행해야 하는지 알고 있는 것이 중요합니다.

### 기본 동작: 접근 제어 건너뜀

기본적으로 **로컬 API 작업은 접근 제어를 건너뜁니다**. 이를 통해 시스템이 현재 사용자가 적절한 권한을 가지고 있는지 확인하지 않고 작업을 실행할 수 있습니다. 이는 사용자 컨텍스트가 작업 수행에 필요하지 않은 관리자나 서버 측 스크립트에서 도움이 될 수 있습니다.

#### 예를 들어:

```ts
// 이 작업의 접근 제어는 기본적으로 건너뛰어집니다
const test = await payload.create({
  collection: 'users',
  data: {
    email: 'test@test.com',
    password: 'test',
  },
})
```

### 접근 제어 존중하기

접근 제어를 존중하고 사용자가 적절한 권한을 가진 경우에만 작업이 수행되도록 하려면, 명시적으로 `user` 객체를 전달하고 `overrideAccess` 옵션을 `false`로 설정해야 합니다.

- `overrideAccess: false`: 이는 접근 제어가 **건너뛰어지지 않도록** 하고 작업이 현재 사용자의 권한을 존중하도록 합니다.
- `user`: 인증된 사용자 컨텍스트를 작업에 전달합니다. 이는 시스템이 사용자가 작업을 수행할 적절한 권한을 가지고 있는지 확인하도록 합니다.

```ts
const authedCreate = await payload.create({
  collection: 'users',
  overrideAccess: false, // 이는 접근 제어가 적용되도록 합니다
  user, // 권한을 확인하기 위해 인증된 사용자를 전달
  data: {
    email: 'test@test.com',
    password: 'test',
  },
})
```

이 예시는 전달한 `user`가 적절한 접근 제어 권한을 가진 경우에만 문서가 생성되도록 허용합니다.
